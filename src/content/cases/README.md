# Кейси — як додати, змінити або прибрати

Ця папка — єдине місце, де живуть кейси. Сайт збирає портфоліо з неї автоматично:
**додали папку з кейсом → кейс з'явився на сайті; видалили папку → зник.**
Жодного коду чіпати не треба.

## Як додати новий кейс

1. Скопіюйте папку `_template` і назвіть її латиницею через дефіс, наприклад `sto-motor-kyiv`.
   Ця назва стане адресою сторінки кейсу: `/cases/sto-motor-kyiv/`.
2. Відкрийте `case.yaml` у новій папці та заповніть усі поля (пояснення — нижче і в самому файлі).
3. Покладіть фото поруч із `case.yaml` у цій самій папці:
   - `cover.jpg` — головне фото (краще горизонтальне, реальний бізнес, не макет);
   - `owner.jpg` — фото власника (квадратне або портретне);
   - додаткові фото — як завгодно, головне вказати їхні імена в полі `gallery`.
4. Готово. Кейс з'явиться на головній (укр. і англ.) і отримає власну сторінку в обох мовах.

## Як прибрати кейс

- **Тимчасово сховати:** у `case.yaml` поставте `published: false`.
- **Видалити назавжди:** видаліть усю папку кейсу.

## Поля у `case.yaml`

| Поле | Що це |
|---|---|
| `published` | `true` — показувати, `false` — сховати |
| `order` | Порядок на головній: 1 — перший (ставте найсильніший кейс) |
| `business.name` | Назва бізнесу, як її показувати (можна з лапками) |
| `business.niche` | Ніша: `ua` — українською, `en` — англійською |
| `business.city` | Місто, двома мовами |
| `liveUrl` | Посилання на живу сторінку, яку ви зробили |
| `owner.name` | Ім'я власника |
| `owner.photo` | Файл фото власника, напр. `./owner.jpg` |
| `cover` | Головне фото кейсу, напр. `./cover.jpg` |
| `gallery` | Список додаткових фото (може бути порожнім: `[]`) |
| `problem` | З чим прийшов бізнес — 2–4 речення, двома мовами |
| `solution` | Що ви зробили: хід думки і рішення, не «наліпив кнопки» |
| `result` | Результат СЛОВАМИ ВЛАСНИКА, коротко: «Записані на два тижні вперед» |
| `testimonial.quote` | Повна цитата-відгук. `en` — переклад оригіналу |
| `testimonial.originalLang` | Мова оригіналу відгуку: `ua` або `en` |
| `testimonial.video` | ID відео з YouTube (те, що після `watch?v=`) або порожньо `""` |

## Правила, щоб кейс продавав

- Результат пишіть словами власника, а не відсотками: «почали дзвонити з сайту», а не «конверсія +40%».
- Фото — справжній бізнес і справжня людина, не скриншот макета.
- В англійських полях — не дослівний переклад, а та сама думка природною англійською.
- Довгі тексти пишіть через `|` з відступом (як у шаблоні) — так YAML розуміє кілька рядків.

Якщо після змін сайт не збирається — найчастіше пропущене поле або зайвий відступ.
Повідомлення про помилку назве файл і поле, яке треба виправити.
